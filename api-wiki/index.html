本文以当前参与构建的源码为准。进度文件中的旧版本条目是历史实验记录，不代表当前构建仍包含对应实现。

快速开始

print('CS2Lua API', CS2LUA_API_VERSION)
events.render:set(function()
    -- render callback
end)

events.name:set 是单槽注册；再次设置同一事件会替换此前回调。

network

network.get(url [, headers], callback)
network.post(url, data [, headers], callback)
-- callback(response, status)

WinHTTP 后台执行，回调在 Lua tick 派发；请求成功入队返回 true。POST table 自动 JSON 编码。

db 当前只保存在 Lua 状态内，卸载/重载即清空。空 Lua table 默认编码为 JSON 数组 []，无法自动表达空对象 {}。

panorama 实验

Panorama 当前不是稳定 API。 V8 Fatal Assertion 会直接终止游戏，Lua pcall 无法捕获。

推荐模式：一次性 JS-only

local ok, err = panorama.run_script([[
(() => {
    if (globalThis.__my_script) return;
    globalThis.__my_script = true;
    $.RegisterForUnhandledEvent('ShowAcceptPopup', () => {
        $.Msg('popup'); // 逻辑留在 JS 内
    });
})();
]], 'CSGOMainMenu')

不要这样使用

  • 在 render/tick/timer 中持续调用 loadstring/open 代理。
  • 跨帧保存 JS Array/Object/Function/PersistentProxy。
  • Lua 轮询 JS 事件队列。
  • JS 回调 Lua，再从 Lua 重入 Panorama。
  • 面板销毁后继续调用旧 Context 的代理。
-- 实验接口
panorama.loadstring(js [, panel])
panorama.loadrawstring(js [, panel])
panorama.open([panel])
panorama.run_script(js [, panel])
panorama.runScript(js [, panel [, xml_context]])
panorama.hasPanel(name)
panorama.getPanel(name [, fallback])
panorama.debug_status()

当前没有正式的 panorama_events API。测试版采用 JS-only 事件所有权。
